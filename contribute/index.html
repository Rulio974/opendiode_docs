# Comment contribuer

Il y a de nombreuses façons de contribuer à l'avancement et l'amélioration du projet, et nous apprécions toutes les contributions, qu'elles soient petites ou grandes.

Nous contacter

Si vous avez des questions ou des suggestions, n'hésitez pas à nous contacter par mail : baril@et.esiea.fr

# Rapporter des bugs et contribuer au code

Si vous trouvez un bug, veuillez ouvrir une issue (opens new window). Assurez-vous de fournir autant d'informations que possible pour que nous puissions reproduire le bug. Si vous avez trouvé une solution, n'hésitez pas à soumettre une pull request (opens new window).

# Contribuer à la documentation

La documentation est un élément essentiel du projet. Elle permet aux utilisateurs de comprendre comment installer, utiliser et maintenir la solution. Si vous trouvez des erreurs ou des incohérences, ou si vous avez des suggestions pour améliorer la documentation, n'hésitez pas à ouvrir une issue (opens new window) ou à soumettre une pull request (opens new window).
