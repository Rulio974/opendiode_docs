# Première utilisation

# Connexion aux machines

Déconnectez vous de l'utilisateur de root et reconnectez vous avec l'utilisateur opendiode et le mot de passe définit pendant l'installation.

Si l'installation s'est bien passée, vous devriez arriver sur l'interface d'administration d'OpenDiode.

Options Description
1 - 4 Modifier les mots de passe des utilisateurs de chacune des VMs
5 Configurer l’hôte
6 Eteindre le système
7 Ouvrir un shell

# Changer les mots de passe et se connecter en SSH

Commencez par changer les mots de passe utilisateur et root des machines INPUT, Firewall 1 (Netfilter) et OUTPUT.

Changer mot de passe

L'interface d'administration d'OpenDiode vous permet de changer les mots de passe des machines virtuelles grâce aux options de 1 à 4.

Testez ensuite de vous connecter aux machines via SSH.

  • Machine INPUT :
ssh share@10.101.0.1
  • Machine OUTPUT :
ssh user@10.104.0.1
  • Machine Netfilter :
ssh share@10.102.0.1

PfSense

Vous vous connecterez à l'interface PfSense plus tard

# Configuration de l'hôte

La configuation de l'hôte est accessible via l'option 5 de l'interface d'administration.

Options Description
1 Modifier l’adresse IP de son interface d’administration
2 Modifier le port d’écoute de son serveur SSH (22 par défaut)
3 - 4 Modifier les mots de passe des utilisateurs (opendiode et root)
5 Effacer les logs
6 Retourner à l'interface principale

Logs

Si l’administrateur souhaite effectuer une action plus spécifique, qui n’est pas possible depuis cette interface d’administration (lire les logs, accéder aux VMs via SSH), il est toujours possible d’ouvrir un shell (Bash) depuis le menu principal avec l’option n°7.

Déconnexion

Pour se déconnecter depuis l’interface d’administration, il suffit de presser les touches Ctrl+C.

# Accès aux journaux (logs)

Pour accéder aux journaux, il suffit de se connecter en SSH à l'hôte et de se rendre dans le dossier /var/log/opendiode.


cd /var/log/opendiode

Ils sont constitués de 3 fichiers :

  • install.log : Relatifs à l’installation d’OpenDiode.
  • run.log : Produits au démarrage de la diode
  • stop.log : Générés à l’extinction de la diode

# Transfert de données

La diode est accessible sur le réseau d’entrée à l’adresse 10.0.1.12. Sur le réseau de sortie, son adresse est 10.0.2.11.

Actuellement, 3 protocoles sont activés en entrée :

Protocole Port
FTP 21
SSH 22
SYSLOG 514 TCP + UDP

Les protocoles FTP et SSH sont accessibles avec l’utilisateur share dont le mot de passe a été par l’administrateur (VM1).

Le serveur Syslog ne nécessite pas d’authentification pour recevoir des logs.

Les données peuvent être récupérées depuis le réseau de sortie via les protocoles FTP et SSH, via l’utilisateur share dont le mot de passe a également été initialisé par l’administrateur lors de l’installation (VM4).
