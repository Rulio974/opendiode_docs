# Utilisation virtualisée

Utilisation non recommandée

Cette installation n'est actuellement pas recommandée. Elle ne permet pas la gestion des différentes VMs, l'accès à l'interface d'administration et aux logs.

# Première utilisation

# Connexion aux machines

Lancez les 4 machines virtuelles et connectez vous avec les identifiants suivants :

VM1 (input) et VM4 (output) :

  • Utilisateur : share / Mot de passe : share.
  • Super utilisateur : root / Mot de passe : toor.

VM2 (Netfilter) :

  • Utilisateur : user / Mot de passe : user.
  • Super utilisateur : root / Mot de passe : toor.

VM3 (PfSense) :

  • Utilisateur : admin / Mot de passe : pfsense.

# Transfert de données

La diode est accessible sur le réseau d’entrée à l’adresse 10.0.1.12. Sur le réseau de sortie, son adresse est 10.0.2.11.

Actuellement, 3 protocoles sont activés en entrée :

Protocole Port
FTP 21
SSH 22
SYSLOG 514 TCP + UDP

Les protocoles FTP et SSH sont accessibles avec l’utilisateur share dont le mot de passe a été par l’administrateur (VM1).

Le serveur Syslog ne nécessite pas d’authentification pour recevoir des logs.

Les données peuvent être récupérées depuis le réseau de sortie via les protocoles FTP et SSH, via l’utilisateur share dont le mot de passe a également été initialisé par l’administrateur lors de l’installation (VM4).

# Accès aux interfaces web

Accès aux interfaces web

Vous pouvez dans le cadre d'une démonstration accéder aux interfaces web des machines virtuelles et télécharger des fichiers depuis le réseau d'entrée et les visualiser depuis le réseau de sortie.

# Préparation

Tout d'abord, dans VirtualBox, ajoutez une 4ème interface réseau pour les VMs INPUT (VM1) et OUTPUT (VM4).

Elle devra être configurée en mode Accès par pont et être connectée à votre carte réseau physique.

Redémarrez ensuite les VMs.

# Vérifications

Sur les VMs INPUT (VM1) et OUTPUT (VM4), connectez vous en tant que root vérifiez que les serveurs web apache2 sont bien lancés :


systemctl status apache2

Si ce n'est pas le cas, lancez les services :


systemctl start apache2

Trouvez et activez la carte réseau nouvellement ajoutée :


ip a

Trouvez le nom de la carte réseau, ici enp0s10.


ip link set enp0s10 up

Donnez une adresse IP à la carte réseau :


dhclient enp0s10

# Accès

Connectez vous à l'interface web de la VM INPUT (VM1) à l'adresse http://@IP_INTERFACE_WEB_VM1 et téléchargez un fichier.

De même, connectez vous à l'interface web de la VM OUTPUT (VM4) à l'adresse http://@IP_INTERFACE_WEB_VM1 et visualisez le fichier téléchargé.
